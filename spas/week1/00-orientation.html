Week 1 Orientation

Estimated time: ~15 minutes  |  Prerequisites: None

The Gap Between Voltage and This Webpage

Every program you have ever written exists inside your computer as nothing more than patterns of electrical voltage. A wire either has voltage on it, or it doesn't. Two states. That is the entire physical vocabulary of a digital computer.

Yet the computer you are sitting at right now can display this sentence in color, play video, run a web browser with dozens of tabs, and compile code you write in Python, Java, or C. How is that possible when the underlying hardware only understands ON and OFF?

The Gap

What you know: You can count, add, and represent any number using the ten digits (0–9) you've used since childhood. You know what a "byte" is, at least vaguely, and you've seen "hex" in color codes or memory addresses.

What you can't do yet: Explain how a machine that only understands "voltage present" or "voltage absent"—two states—can represent the number 2,731, the letter 'M', or the color blue.

What you will be able to do by the end of this week: Take any integer, character, or simple data value and show exactly how it is stored as a sequence of 0s and 1s in computer memory. More importantly, you will understand that the same sequence of bits can mean completely different things depending on how the computer interprets them—which is the single most important idea in this entire course.

If you start SPA 4 first, complete SPA 3 before reaching the synthesis section.

Tools You'll Use in This Course

Throughout this course you will use two primary tools to build and simulate digital circuits and assembly programs. No installation is needed this week—you will encounter these tools starting in Weeks 2 and 5 respectively.

🔌
Logisim Classic 2.7.1
Starting Week 2, you will build digital circuits from primitive logic gates (AND, OR, NOT, NAND, etc.). By the end of the course, you will have constructed a complete CPU simulation—all within a single .circ file. Hosted for download on the course page.
📟
MARS Simulator
Starting Week 5, you will write MIPS assembly programs that run on a simulated processor. MARS provides an assembler, debugger, and memory viewer so you can see exactly how your code maps to machine-level operations.
📖
Saylor Academy — CS301: Computer Architecture
Your primary reference textbook throughout the course. Free, openly licensed (CC BY 3.0), and organized into nine units that map to our four-phase progression. When SPAs reference outside reading, you will log in and navigate to specific units as directed. Open Saylor CS301 →

How This Course Works

Accessing Course Materials

All primary learning content is delivered through Single-Page Applications (SPAs) like this one. Each SPA contains prose, interactive widgets, and explicit references to the Saylor CS301 textbook. Navigate through SPAs in the order presented on the sidebar.

Submitting Assignments

Assignments include Logisim circuit checkpoints, MIPS programming exercises, and research papers. Submission instructions and deadlines are listed in the full syllabus. All Logisim work is contained within a single .circ file that grows each week—never create a new file.

Weekly Effort Expectation

This is a 3-unit course requiring approximately 9 hours per week:

  • 6 hours out-of-class — SPA content, assignments, textbook reading

  • Midterm Exam (Week 8) — Covers Weeks 1–8, Phases 1 and 2.

Where Week 1 Fits in the Full Course

This course is built as a single continuous progression—not a sequence of disconnected topics—running from raw voltage to a complete, working CPU simulation. Week 1 is the first step on that ladder.

"Every program you have ever written exists inside your computer as nothing more than patterns of electrical voltage. This week, you learn the rules that make that possible."
Week 1 — Binary & Number Representation
Bits, bytes, bases, signed/unsigned, floating-point, character encoding, memory layout
Weeks 2–4 — Digital Logic
Logic gates → combinational circuits → sequential circuits / flip-flops
Weeks 5–8 — Assembly Language & Hardware
Registers, ALU, RAM, MIPS programming (interleaved hardware + assembly tracks)
Weeks 9–12 — CPU Design Theory
Von Neumann model, fetch-decode-execute, control unit, datapath integration
Weeks 13–17 — Integration & Modern Topics
Complete CPU build, pipelining, cache, multicore, final project
The Thread That Connects Everything

Every new concept in this course is introduced with three elements:

  1. "Here's what we have so far" — anchored to the concept you just learned
  2. "Here's the problem we can't solve yet" — creating the need for the next concept
  3. "Here's what we can now do" — closing the loop and showing forward progress

No module will feel disconnected. Every step is a necessary rung on a single ladder from raw voltage to a working computer.

Ready to Begin?

When you are ready, start with SPA 1: Why Binary? — where we begin at the physical foundation: the two voltage states that make everything possible.
